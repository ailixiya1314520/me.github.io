Niagara Fire Effect

This project demonstrates a real-time fire and smoke particle system built with Unreal Engine Niagara. The effect includes dynamic lighting integration, wind interaction, and GPU-accelerated simulation.

Technical Implementation

The particle system is built using a custom C++ module that interfaces with Niagara's data-oriented framework. Below is the core update loop for the fire simulation.

23        Particle.Color = FLinearColor::Lerp(

26    }

Shader Implementation

The NPR rendering is achieved through a custom shader that modifies the lighting calculation to produce the cel-shaded look.

2float3 CalculateNPRLighting(float3 Normal,


3    float3 LightDir, float3 ViewDir) {


4


5    // Diffuse calculation with cel-shading


6    float NdotL = saturate(dot(Normal, LightDir));

8    // Apply color banding for cel-shaded look

10    


11    // Rim lighting for stylized effect

13        saturate(dot(ViewDir, reflect(-LightDir, Normal))),


14        8.0


15    );


16    


17    return NdotL * float3(1.0, 0.9, 0.8) + Rim * float3(0.0, 0.8, 1.0);


18}

Performance Optimization

To maintain high frame rates, the system uses several optimization techniques including:

  • GPU instancing for particle rendering
  • Distance-based LOD system
  • Burst compilation for C++ hot paths
  • Texture array for efficient memory access

Perlin Noise Integration

The fire effect uses 3D Perlin Noise to create natural-looking turbulence. This procedural noise is sampled at each particle's position and scaled by the turbulence parameter to add organic movement.

The noise frequency determines the level of detail in the fire's chaotic behavior, while the turbulence factor controls the overall intensity of the displacement.

  • 3D noise sampling for temporal coherence
  • Scalable noise frequency
  • Wind-adaptive turbulence
